Форма произвольной регистрации

С помощью этой формы можно произвести регистрацию либо отмену регистрации произвольно различными способами.

 При открытии формы в неё передается список метаданных в которых необходимо произвести поиск объектов (выделенные строки основной формы).

Отбор по значениям реквизитов

Отбор по значениям реквизитов построен на базе системы компоновки данных. В отбор включаются все реквизиты исходного списка метаданных, таким образом имеется возможность отобрать, например, все документы конфигурации за определенный период, имеющий реквизит "Организация", заполненный в соответствии с отбором. Строки отбора можно группировать по условиям И, ИЛИ или НЕ, получая таким образом практически любые варианты отбора.

С помощью отбора по значениям реквизитов можно отбирать не только ссылочные объекты метаданных, но и записи регистров, как подчиненных регистратору, так и независимых регистров сведений.

Перенос данных в результирующую таблицу производится командой  "Заполнить данные" командной панели отбора.

При выборе корня конфигурации (т.е. фактическом включении всех метаданных) при открытии формы, отбор по значениям реквизитов блокируется.

 

При выборе большого набора метаданных количество доступных реквизитов в отборе может быть очень велико, что может замедлить работу и привести к сложностям в поиске нужного реквизита. Система компоновки данных подставляет в качестве имени элемента отбора синоним как для обычных, так и для стандартных реквизитов, таким образом, имена реквизитов в отборе могут отличаться от имен в конфигурации. Так, например, если в набор метаданных попал справочник "Валюты" из типовых конфигураций 1С, реквизит "Код" для всех метаданных в отборе будет именоваться "Код числовой".

 

Отбор с помощью произвольного запроса

Отбор с помощью произвольного запроса позволяет выполнить отбор объектов с помощью запроса.

В толстом клиенте в контекстном меню поля "Текст произвольного запроса" доступен конструктор запроса. В тонком клиенте запрос должен быть подготовлен в конфигураторе и перенесен через буфер обмена, либо написан вручную. Необходимые параметры запроса указываются в таблице параметров, расположенной под полем произвольного запроса. Возможно автоматическое заполнение таблицы параметров командой  "Обновить параметры запроса". Перенос данных в результирующую таблицу производится командой "Заполнить данные", расположенной в командной панели таблицы параметров.

Для правильного заполнения результирующей таблицы запрос должен соответствовать следующим условиям:

  • Для ссылочных типов данных результат запроса должен содержать поле "Ссылка", например

ВЫБРАТЬ РАЗРЕШЕННЫЕ
     Контрагенты.Ссылка КАК Ссылка
ИЗ
     Справочник.Контрагенты КАК Контрагенты
ГДЕ
     Контрагенты.Партнер.Поставщик = ИСТИНА

  • Для регистров имеющих подчинение регистратору результат запроса должен содержать поле "Регистратор", заполненное соответствующей ссылкой на документ и поле "ИмяМетаданных", в котором должно быть указано полное имя метаданных регистра. Например:

ВЫБРАТЬ РАЗРЕШЕННЫЕ РАЗЛИЧНЫЕ
     СебестоимостьТоваров.Регистратор КАК Регистратор,
     "РегистрНакопления.СебестоимостьТоваров" КАК ИмяМетаданных
ИЗ
     РегистрНакопления.СебестоимостьТоваров КАК СебестоимостьТоваров
ГДЕ
     СебестоимостьТоваров.Период МЕЖДУ &НачалоПериода И &ОкончаниеПериода

  • Для независимых регистров сведений результат запроса должен содержать поле "ИмяМетаданных", в котором необходимо указать полное имя метаданных регистра, а так же все поля измерений регистра с соответствующими именами, для периодических регистров дополнительно поле "Период". Например:

ВЫБРАТЬ РАЗРЕШЕННЫЕ РАЗЛИЧНЫЕ
     ФИОФизЛиц.Период КАК Период,
     ФИОФизЛиц.ФизЛицо КАК ФизЛицо,
     "РегистрСведений.ФИОФизЛиц" КАК ИмяМетаданных
ИЗ
     РегистрСведений.ФИОФизЛиц КАК ФИОФизЛиц
ГДЕ
     ФИОФизЛиц.ФизЛицо.ДатаРождения > &ДатаРождения

Результирующая таблица "Данные"

В эту таблицу выгружаются объекты отобранные с помощью отбора или произвольным запросом. Кроме этого, объекты в таблицу (только ссылочного типа) можно добавлять с помощью команды  "Добавить". Предусмотрена команда "Подбор", с помощью которой можно подобрать несколько объектов без закрытия формы выбора. Если при открытии форме был передан набор из нескольких метаданных, перед выполнением операции предлагается выбрать из списка конкрентый тип данных. При выборе строки таблицы открывается для просмотра и редактирования:

  • Для ссылочных типов данных - соответствующий ссылке объект.
  • Для записей регистров, подчиненных регистратору - документ-регистратор.
  • Для независимых регистров сведений - форма записи.

Строки в таблице можно сортировать, перемещать и удалять. Выбор строк в таблице множественный.

Установка флажка, расположенного над таблицей позволяет совместно с документами регистрировать (или отменять регистрацию) движений в регистрах.

Операция по регистрации или удалению регистрации объектов запускается командой  "Выполнить".

ВНИМАНИЕ! Никаких дополнительных подтверждений на выполнение действий по регистрации или удалению регистрации не производится.
